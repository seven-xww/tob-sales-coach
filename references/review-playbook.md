# ToB 销售复盘手册

用于通话复盘、会议复盘、聊天记录复盘、proposal 复盘、谈判复盘和销售能力训练。

## 复盘输出

任何销售电话或会议复盘，都先提取客户事实，再复盘销售表现。

必须包含：
- 客户推进卡：更新当前阶段、客户等级、风险和下一步。
- 客户事实提炼：业务问题、影响、当前方案、触发事件、预算信号、决策链、时间线、竞争/替代方案、成功标准、下一步。
- 客户质量评分：10 分制，重点看 ICP、痛点、影响、预算、决策、周期。
- 客户等级：A 重点推进，B 培养推进，C 轻跟进，D 筛掉。
- 成交概率：用区间，不假装精确。
- 用户表现：信任建立、问题深度、价值框架、节奏控制、关键人推进、下一步推进。
- 原话改写：引用关键原话，给更好的替代表达。
- 下一步打法：消息怎么发、电话怎么约、是否约方案会/专家会/关键人会、是否补资料。
- `talk-a-little-bit-more` 模块。

## 事实提取

先提取事实，再给建议：
- 客户是谁、联系人是谁？
- 客户明确说出的业务问题是什么？
- 是什么触发他们现在考虑改变？
- 现在用什么方案或临时办法？
- 问题造成什么影响：成本、收入、风险、效率、体验、合规、增长、人力、时间？
- 谁参与决策、评估、采购、法务、安全、实施和日常使用？
- 出现了什么预算或价格信号？
- 出现了什么时间线或截止节点？
- 出现了什么竞品或替代方案？
- 客户如何判断项目成功？
- 双方约定的下一步是什么？

缺失事实要标为未知，不要编造。

## 销售表现维度

评估：
- 信任建立：销售是否连接了客户背景和自身可信度，而不是过度自卖自夸？
- 需求深度：是否问到影响、根因、当前方案、关键人和成功标准？
- 业务框架：是否把产品/服务连接到业务结果？
- 关键人控制：是否识别最终决策人、影响人、使用者、采购、法务/安全？
- 节奏控制：是否避免过早推销，并拿到清晰下一步？
- 异议处理：是否把表面立场翻译成真实利益？
- 边界控制：是否避免免费咨询、过度定制 proposal 或过早打折？

## talk-a-little-bit-more

每次复盘都必须包含此模块。目标是训练销售在客户回答上多停留一层。

流程：
1. 用 2-3 句话概括对话哪里差点进入更深一层。
2. 找信号：模糊词、大词、跳过流程、情绪变化、突然换话题、“之前试过”“领导关注”“预算紧”“内部流程”“不着急”等。
3. 对每个时刻说明：客户说了什么、销售怎么接、没展开的是什么、如果重来可以多问哪一句。
4. 总结共性：急着介绍、没问影响、回避预算、错过关键人、接受模糊下一步、收口弱。
5. 最后必须只给一个动作：

```markdown
**下次只改一个动作**

[一句具体行为]
```

没有原话时，写“根据你的描述推测”。不要编造引语。

## 常用追问

复盘发现需求挖掘不足时，可使用：
- “这个问题现在对业务影响最大的是成本、效率、收入，还是风险？”
- “现在你们是怎么解决的？为什么这个办法不够用了？”
- “如果半年后这个项目算成功，内部看哪几个指标？”
- “这件事除了您，还有谁会参与判断？”
- “预算这块是已经有范围，还是要先内部立项？”
- “您现在也在看其他供应商吗？主要比较哪几个点？”
- “如果这个月不推进，会有什么影响？”
- “我刚才理解得对不对：您最担心的不是功能，而是上线后团队用不起来？”

## 下一步质量

弱下一步：
- “我发资料您看看。”
- “有需要再联系。”
- “您考虑一下。”
- “后面微信沟通。”

强下一步：
- 有明确负责人。
- 有明确日期/时间。
- 有明确参会人。
- 有明确会议目的。
- 有明确会前资料或准备动作。

示例：
> 我今天发您案例和范围拆解。周三下午 3 点我们约 30 分钟，把业务负责人和技术同事一起拉上，重点确认范围、集成风险和预算路径。
