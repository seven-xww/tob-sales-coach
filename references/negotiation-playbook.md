# 通用 ToB 谈判内参

用于 proposal 阶段谈判、价格异议、采购/法务流程、让步交换、关键人对齐、拖延挽回、拒绝挽回和成交策略。

## 核心原则

- 不和立场打架。“太贵了”“预算不够”“别人便宜”“再看看”都是立场，要翻译成利益：预算恐惧、ROI 不确定、内部交代压力、采购压价、风险担心或试探折扣。
- 把人和问题分开。客户压价不是对销售的不尊重，先承认感受，再回到业务问题、范围、价值、标准、风险和下一步。
- 谈利益，不谈输赢。销售目标是高质量成交、保护范围和专业信任；客户目标是降低风险、方便内部解释、审批更顺、获得确定性。
- 先创造选项，再做选择。不要只剩“原价 vs 打折”。可用范围调整、分阶段上线、POC、年框合同、实施包、付款节奏、案例授权或高层对齐。
- 使用客观标准：范围、用量、集成复杂度、服务等级、实施工作量、支持等级、合规要求、交付周期、团队成本、风险。
- 每一次让步都要交换确定性：签约日期、付款节奏、合同年限、确认范围、决策人参会、案例授权、扩容路径或 POC 转正式合作条件。
- 沟通是长期关系。即使这次不成交，也要保留专业的未来关系入口。

## Proposal 阶段流程

开始前，先确认是否适合提交 proposal：业务问题、决策链、预算匹配、价值认同、采购路径、成功标准和项目事实是否足够。不适合时，先准备诊断问题，不要正式报价。

1. 先确认价值，再谈价格：
   > 价格我们可以沟通，但我想先确认一个前提：这次 proposal 里的范围和您现在要解决的问题，是不是匹配的？

2. 追问为什么觉得贵：
   > 我理解您会评估投入。您觉得高，是因为超出原本预算，还是因为和其他供应商比，感觉服务内容差不多但价格差很多？

3. 要对比语境：
   > 如果方便，您可以把对方大概范围或报价结构告诉我。我不是为了否定别人，是帮您判断我们是不是在比同一件事。

4. 把立场翻译成真实利益：
   - “太贵了” -> 投入不确定、内部解释压力、ROI 不清。
   - “别人便宜” -> 范围、交付深度、风险承担不同。
   - “和领导/合伙人商量” -> 内部说服问题。
   - “采购要压价” -> 流程性价格压力。
   - “法务/安全还没过” -> 风险和责任边界未清。
   - “再考虑” -> 优先级、价值认知或时间线不足。
   - “先出完整方案” -> 想降低决策风险，但可能越过合作边界。

## 折扣和让步交换

任何让步前，先问：
- 是否认可解决方向和范围？
- 如果价格或条款解决，什么时候能确认合作？
- 所有最终决策人是否已经对齐？
- 客户能交换什么确定性：签约日期、付款节奏、合同年限、确认范围、高层会议、案例授权、扩容路径或 POC 转正条件？

基础话术：
> 我可以帮您沟通商务空间，但我不能只跟公司说“客户觉得贵”。我需要知道真实卡点，以及如果沟通到位，咱们能不能把合作时间和范围定下来。

更强势一点：
> 商务政策一定是跟确定性绑定的。如果只是先探最低价，公司很难给空间。您如果这周能定范围和流程，我就按这个方向帮您争取。

底线/边界话术：
> 这个价格已经是基于当前范围和服务质量能沟通到的边界。再往下不是简单少赚一点，而是会影响团队投入、交付质量或服务范围。如果还要调整，我们需要重新看范围，而不是在同一套内容上继续降。

让步记录：
- 客户要求：客户想改什么。
- 我方条件：客户要用什么确定性或范围调整来交换。
- 边界：底价、范围、SLA、实施、支持、知识产权、数据、法务责任、付款节奏。
- 下一步确认：谁确认、什么时候确认，通过电话、会议、采购还是法务流程确认。

不要建议无条件纯降价。

## 采购、法务和安全

进入采购/法务/安全流程时：
- 识别每个流程负责人和他们关心的问题。
- 要清单、时间线、所需材料、合同模板、安全问卷、供应商准入步骤和付款流程。
- 继续绑定业务方，不要让对话只剩采购。
- 把采购/法务问题翻译成业务影响和风险边界，不要变成人际对抗。

可用话术：
> 采购和法务流程我们配合。为了不让项目卡住，我想同步确认一下：业务方这边的合作方向已经认可了吗？如果流程上有材料或条款问题，我们逐项处理。

## 多关键人处理

客户说要内部讨论时，先问：
> 内部主要会关注什么？预算、效果、实施风险、系统兼容、合同条款，还是上线时间？

推动联合会议：
> 要不我们把关键人一起约 20 分钟。我怕您一个人转述，会把业务价值压缩成一个价格，反而不好推进。

如果业务发起人必须内部转述，给三条可复制信息：
- 这解决的是 [业务问题]，不只是买一个产品/服务。
- 报价对应范围、交付深度、风险控制和支持等级。
- 只按最低价选择，可能带来更高的返工、延期、使用率、合规或机会成本。

## 拖延和沉默

客户 proposal 后沉默时：
- 不要只问“看了吗”或“考虑得怎么样”。
- 把姿态调整成帮助客户做内部决策。
- 主动提出把 proposal 拆成必做项、可选项和后续阶段。
- 询问卡点是预算、内部对齐、采购、竞品比较、风险还是时间。

消息话术：
> 我这边不催您定，只是想确认 proposal 里有没有哪一块需要我帮您拆开说明。如果内部在看预算，我可以帮您拆成“必须先做的部分”和“后续可扩展部分”，方便您跟团队沟通。您今天或明天方便，我用 10 分钟电话帮您过一下？

## 拒绝挽回

客户明确拒绝时：
> 明白，那这次我就不继续打扰您了。不过我想请教一句，您最后没选我们，主要是预算、服务范围、内部优先级，还是信任感上还有顾虑？您直接说就行，也方便我后面改进。

项目暂停时：
> 明白，项目节奏调整很正常。我想确认一下，这次暂停主要是预算收紧、内部优先级变化、采购流程卡住，还是业务项目本身延后？我这边后续跟进节奏也会放轻一点。

## 谈判请求的必备输出

用户请求谈判帮助时，输出：
- 当前谈判诊断：真实预算不足、比价压力、风险担心、内部未对齐、采购/法务/安全问题或拖延。
- 本轮目标。
- 不可让步边界：底价、范围、知识产权、数据、SLA、交付质量、法务责任、团队时间。
- 可交换条件：签约时间、付款、合同年限、决策人参会、确认范围、案例授权、POC 转正、扩容路径。
- 如果正在谈让步，输出让步记录。
- 电话话术和文字消息话术。
- 下一步动作。
